# 教学使用手册（原型版）

## 适合场景

- 小学科学、初中物理、高中物理的探究性作业。
- 科技节、研学项目、跨学科主题活动。
- 学生自主探究 + 小组展示。

## 一节课 40 分钟怎么安排

| 时间 | 环节 | 学生活动 |
|------|------|---------|
| 5 min | 引入 | 教师展示一个示例动画，引出问题。 |
| 10 min | 选题与拆解 | 学生 2-3 人一组，选择一个问题，填写实验模板。 |
| 15 min | AI 生成与调试 | 学生生成 Prompt，交给 AI 生成代码，在 p5.js Web Editor 运行。 |
| 8 min | 观察与记录 | 调节参数，截图或录屏记录现象。 |
| 2 min | 分享 | 1-2 组快速展示。 |

## 推荐选题（小初高）

| 学段 | 示例问题 |
|------|---------|
| 小学 | 影子为什么会变长变短？ |
| 小学 | 彩虹是怎么出现的？ |
| 初中 | 抛物线运动与初速度、角度的关系 |
| 初中 | 光线折射：水中的筷子为什么“弯”了？ |
| 高中 | 电场线的分布与电荷位置 |
| 高中 | 单摆周期与摆长、重力加速度的关系 |

## 评价建议

重点评价“探究过程”而不是代码完美度：

- 是否能清晰描述要探究的问题？
- 是否能正确识别科学概念和可调控变量？
- 是否能通过动画观察到现象并解释原因？
- 是否能与同学分享发现和疑问？

## 常见课堂问题

**学生不知道问什么？**
建议提前准备 10 个“问题卡片”供抽签选择。

**代码运行出错？**
让学生把错误信息复制给 AI；教师可准备一个“通用修复 Prompt”：

```text
请修复以下 p5.js 代码的错误，并保持原有功能：
[paste code here]
```

**动画太简单？**
鼓励学生增加参数滑块、轨迹保留、颜色变化等。
